# Wolf Brainstorm - 目标澄清

## 用法

\`\`\`bash
/wolf-brainstorm              # 开始头脑风暴
\`\`\`

## 功能

### 1. 目标澄清

系统不断提问，直到目标清晰：

\`\`\`
🐺 Wolf Pack 头脑风暴
═════════════════════════════

第 1 问: 目标描述
─────────────────────────────
请描述你想要完成的目标：
> [用户输入]

好的，你想要：[用户输入总结]

让我确认一些细节...
\`\`\`

### 2. 问题清单

1. **目标描述** - "你想要完成什么？"
2. **背景上下文** - "为什么需要这个？当前情况是什么？"
3. **约束条件** - "有什么限制或约束？" (时间、资源、技术等)
4. **验收标准** - "如何知道目标达成了？"

### 3. 清晰度评估

系统评估目标清晰度（各维度 0-20 分）：

\`\`\`
📊 目标清晰度评估
─────────────────────────────
✅ 目标明确:    [评分 0-20]
✅ 背景清晰:    [评分 0-20]
✅ 约束明确:    [评分 0-20]
✅ 验收可验证:   [评分 0-20]

总分: [X]/80

[X>=64] 目标清晰，可以继续
\`\`\`

### 4. 输出摘要

\`\`\`
📋 计划摘要
─────────────────────────────

目标: [一句话目标]
验收: [验收标准]
约束: [主要约束]
预估: [X] 小时

接受？(y/n)
\`\`\`

### 5. 记录决策

确认后自动调用 `/wolf-remember decision` 记录决策。

### 6. 执行集成

确认摘要后，系统自动执行以下步骤：

**步骤 1**: 记录决策
```
🐺: 嗷~~

✅ 决策已记录到 rules/decisions.md
```

**步骤 2**: 询问是否需要拆分任务
```
─────────────────────────────────────────────
是否需要将任务拆分为可执行的单元？
  - 输入 'y' 或 '是' → 自动拆分并存储
  - 输入 'n' 或 '否' → 跳过，稍后手动触发
```

**步骤 3** (如果用户确认):
- 自动调用 `/wolf-board --breakdown --save`
- 存储任务文件到 `.wolf/tasks/YYYY-MM-DD-{slug}.units.md`

**步骤 4**: 显示后续操作
```
🐺: 嗷~~
════════════════════════════

✅ 决策已记录到 rules/decisions.md
✅ 任务已拆分并存储到 .wolf/tasks/

📋 已生成任务文件:
  .wolf/tasks/2026-02-13-user-auth.units.md

后续操作:
  - 查看任务: cat .wolf/tasks/2026-02-13-user-auth.units.md
  - 执行任务: /wolf-pack unit-auth-model unit-auth-service
  - 重新拆分: /wolf-board --breakdown
```

### 手动触发

如果用户选择不拆分，可以稍后手动触发：
```
🐺: 嗷~~

💡 提示: 稍后可手动拆分任务
  /wolf-board --breakdown
```
