# Wolf Brainstorm - 目标澄清

## 用法

```bash
/wolf-brainstorm              # 开始头脑风暴
```

## 功能

### 1. 目标澄清

使用 `AskUserQuestion` 工具进行交互式提问：

```
🐺 Wolf Pack 头脑风暴
═════════════════════════════

第 1 问: 核心目标
─────────────────────────────
[弹窗] 你想要完成什么？

选项:
  □ 开发新功能 - 从零开始开发一个完整的功能模块
  □ 修复 Bug - 解决现有代码中的问题
  □ 重构代码 - 改进现有代码结构
  □ 其他 - 请在备注中详细说明
```

### 2. 问题清单

1. **核心目标** - "你想要完成什么？"
2. **约束条件** - "有哪些约束条件？" (可多选)
3. **排除范围** - "哪些功能/需求不在范围内？"
4. **验收标准** - "如何判断目标已完成？" (可多选)

### 3. 清晰度评估

系统评估目标清晰度（各维度 0-20 分）：

```
📊 目标清晰度评估
─────────────────────────────
✅ 目标明确:    [评分 0-20]
✅ 背景清晰:    [评分 0-20]
✅ 约束明确:    [评分 0-20]
✅ 验收可验证:   [评分 0-20]

总分: [X]/80

[X>=64] 目标清晰，可以继续
```

### 4. 输出摘要

```
📋 计划摘要
─────────────────────────────

目标: [一句话目标]
验收: [验收标准]
约束: [主要约束]
预估: [X] 小时

[弹窗] 确认计划
  □ 确认并记录 - 记录到决策记忆并询问是否拆分任务
  □ 需要修改 - 重新进入提问流程
```

### 5. 记录决策

确认后自动调用 `/wolf-memory save decision` 记录决策。

### 6. 执行集成

确认摘要后，系统自动执行以下步骤：

**步骤 1**: 记录决策
```
🐺: 嗷~~

✅ 决策已记录到 rules/decisions.md
```

**步骤 2**: 询问是否需要拆分任务
```
[弹窗] 是否需要将任务拆分为可执行的单元？
  □ 是 - 自动拆分并存储
  □ 否 - 跳过，稍后手动触发
```

**步骤 3** (如果用户确认):
- 自动调用 `/wolf-board --breakdown --save`
- 存储任务文件到 `.wolf/tasks/YYYY-MM-DD-{slug}.units.md`

**步骤 4**: 显示后续操作
```
🐺: 嗷~~
════════════════════════════

✅ 决策已记录到 rules/decisions.md
✅ 任务已拆分并存储到 .wolf/tasks/

📋 已生成任务文件:
  .wolf/tasks/2026-02-20-user-auth.units.md

后续操作:
  - 查看任务: cat .wolf/tasks/2026-02-20-user-auth.units.md
  - 执行任务: /wolf-pack unit-auth-model unit-auth-service
  - 重新拆分: /wolf-board --breakdown
```

### 手动触发

如果用户选择不拆分，可以稍后手动触发：
```
🐺: 嗷~~

💡 提示: 稍后可手动拆分任务
  /wolf-board --breakdown
```

## 交互方式

- **弹窗选择** - 使用 `AskUserQuestion` 工具显示选项
- **多选支持** - 约束条件和验收标准支持多选
- **备注输入** - 选择"其他"时可输入详细说明
- **自动评分** - 根据选择自动计算清晰度评分

## 评分阈值

| 总分 | 状态 | 操作 |
|------|------|------|
| 64+ | 清晰 | 生成摘要并确认 |
| 48-63 | 基本清晰 | 建议补充，可继续 |
| <48 | 不够清晰 | 继续提问或强制继续 |
