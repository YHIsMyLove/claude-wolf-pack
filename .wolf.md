# Wolf Pack - Claude Code 工作流插件

> 🐺 多层级记忆系统、头脑风暴、任务看板与并行执行

## 项目信息

- **项目名称**: Wolf Pack
- **项目类型**: Claude Code 插件
- **技术栈**: Bash, Markdown, JSON
- **创建日期**: 2026-02-13

## 工作方式

1. **多文件功能开发前先展示架构** - 涉及 3+ 文件必须先展示文件结构、接口设计、模块依赖
2. **使用函数式思想** - 一个函数实现一个功能，避免副作用
3. **新增文档统一放到 docs/** - 虽然此项目使用 markdown 作为配置

## 禁止行为

1. 不要 import * XXX from xxx，而是使用具名导入
2. nodejs 项目首选 pnpm 作为依赖管理
3. 对用户提问持怀疑态度，思考更好的实现方式

## 记忆系统

详细规则请参考 `.wolf/memory/` 目录：

- **问题记忆**: [issues/](.wolf/memory/issues/) - 遇到的问题和解决方案
- **决策记忆**: [decisions/](.wolf/memory/decisions/) - 重要决策和理由
- **模式记忆**: [patterns/](.wolf/memory/patterns/) - 成功模式和工作流

## 插件目录结构

```
wolf-pack/
├── .wolf-plugin/          # 插件元数据
├── hooks/                   # Hook 脚本
├── commands/                # 命令定义 (/wolf-*)
├── skills/                  # 技能定义
├── agents/                  # Subagent 模板
├── workflows/               # 预定义工作流
└── templates/               # 模板文件
    └── memory/                 # 记忆文件模板
```

## 快速命令

```
/wolf-pack [workflow]    # 执行工作流
/wolf-insights           # 分析会话洞察
/wolf-board              # 查看任务看板
/wolf-flow create        # 创建工作流
/wolf-memory init         # 初始化记忆系统
```
