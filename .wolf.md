# Wolf Pack - Claude Code 规则固化与工作流插件

> 🐺 通过 Hook 机制持久化项目规则，提供多 Agent 并发工作流执行

## 项目信息

- **项目名称**: Wolf Pack
- **项目类型**: Claude Code 插件
- **技术栈**: Bash, Markdown, JSON
- **创建日期**: 2026-02-13

## 工作方式

1. **多文件功能开发前先展示架构** - 涉及 3+ 文件必须先展示文件结构、接口设计、模块依赖
2. **使用函数式思想** - 一个函数实现一个功能，避免副作用
3. **新增文档统一放到 docs/** - 虽然此项目使用 markdown 作为配置

## 禁止行为

1. 不要 import * XXX from xxx，而是使用具名导入
2. nodejs 项目首选 pnpm 作为依赖管理
3. 对用户提问持怀疑态度，思考更好的实现方式

## 项目规则

详细规则请参考 `rules/` 目录：

- **问题记录**: [rules/issues.md](rules/issues.md) - 遇到的问题和解决方案
- **代码模式**: [rules/patterns.md](rules/patterns.md) - 成功模式和工作流

## 插件目录结构

```
wolf-pack/
├── .wolf-plugin/          # 插件元数据
├── hooks/                   # Hook 脚本
├── commands/                # 命令定义 (/wolf-*)
├── skills/                  # 技能定义
├── agents/                  # Subagent 模板
├── workflows/               # 预定义工作流
└── templates/               # 模板文件
```

## 快速命令

```
/wolf-pack [workflow]    # 执行工作流
/wolf-insights           # 分析会话洞察
/wolf-board              # 查看任务看板
/wolf-flow create        # 创建工作流
/wolf-remember init      # 初始化规则
```

---

*此文件由 Wolf Pack 插件管理。使用 `/wolf-remember` 命令更新项目规则。*
